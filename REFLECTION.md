# Reflection

## 1. Can I explain what my code does?

My project is a Trivia Quiz game. The user enters their name, answers 10 questions, and submits the form. My JavaScript fetches trivia questions from an API, displays them on the page, checks the user’s answers, and calculates a score. The score is then saved in localStorage and shown in a table. HTML builds the structure, CSS styles the page, and JavaScript handles the logic and data.



## 2. What was my coding process?

I started by building the HTML layout with the form, question container, and score table. Then I added CSS to style everything. After that, I wrote JavaScript functions step by step. First I worked on fetching questions from the API, then displaying them, then calculating the score. Finally, I added saving scores and displaying them. I used console.log and the debugger to test each part and committed my code often.



## 3. What challenges did I have?

The hardest part was working with the API and making sure the questions loaded correctly. I also had trouble understanding how to loop through the questions and display them properly. Another challenge was calculating the score using the selected answers. At first, my score was not correct, but I fixed it by checking for the data-correct attribute. I also had to figure out how to store and retrieve data using localStorage.



## 4. What would I do differently now?

I would test each function earlier before moving on to the next part. This would help me catch errors faster. I would also plan my JavaScript logic more before coding. Another thing I would do is use the debugger sooner instead of only console.log, because it makes it easier to see how the program runs step by step.
